**版本**: v1.1

**产品定位**: 面向中文网文作者的AI写作质量与商业化辅助SaaS平台

## 第一章 为什么做 — 问题分析与立项依据

### 1.1 创作者的真实困境

当前网文作者在使用AI辅助创作时，面临一个核心矛盾：**AI工具能帮你"写出字"，但无法帮你"写出能赚钱的字"。**

具体表现为以下六大痛点：

**痛点一：逻辑不自洽**
- 人物性格突变、武功名称前后不一致、时间线混乱
- 根因：通用AI无长篇记忆管理能力，写到第50章已忘记第5章的设定

**痛点二：幻觉编造**
- AI"发明"新设定、凭空创造不存在的剧情线、角色关系错乱
- 根因：大模型基于概率生成，缺乏"合同式"的硬约束机制

**痛点三：叙事过度**
- 描写冗长注水、节奏拖沓、该爽的地方不爽
- 根因：AI不懂网文特有的"节奏感"和"爽点密度"标准

**痛点四：设计不符合网文规律**
- 不符合目标平台的题材套路、黄金三章不过关、追读留存差
- 根因：通用AI缺乏网文行业知识，无法评估"追读力"

**痛点五：商业化路径模糊**
- 不知道自己的题材是否有IP改编潜力、不知道哪个平台更适合签约
- 根因：缺乏数据驱动的商业化分析工具

**痛点六：现有工具门槛高**
- webnovel-writer需要Claude Code环境，Dify需要技术能力，星月写作长篇管理弱
- 根因：市场上缺少"开箱即用+质量深度"的中间态产品

### 1.2 市场机会窗口

| 市场指标 | 数据 | 意义 |
|----------|------|------|
| 网文作者规模 | 3269.4万人（2025年） | 巨大的潜在用户池 |
| AI工具使用率 | 67%（2026年） | 市场教育已完成 |
| 网文阅读市场 | 502.1亿元 | 创作者有强烈变现动力 |
| 微短剧IP改编 | 1080亿元，网文IP占比超50% | 新增变现通道 |
| IP改编授权增长 | 同比增长81% | 商业化辅助需求爆发 |

### 1.3 竞争空白识别

经过对8款主流竞品的深度分析，发现以下关键空白：

| 能力维度 | 竞品覆盖度 | 机会评级 |
|----------|-----------|----------|
| AI追读力/爽点预测分析 | 几乎空白（仅webnovel-writer有基础版） | 五星级蓝海 |
| 网文专项写作质量评分 | 低（有泛化工具无网文专项） | 四星级 |
| 商业化辅助（IP潜力评估） | 低（蛙蛙写作有漫剧但无数据分析） | 四星级 |
| 低门槛+深度记忆管理 | 低（webnovel-writer强但门槛极高） | 四星级 |
| AI文本生成/续写 | 红海（星月/蛙蛙/笔灵等已饱和） | 一星级（不进入） |

**立项核心判断**：不做"又一个AI写作生成器"，做"AI写作质量与商业化分析平台"——这是目前市场上无人占据的定位。

## 第二章 做什么 — 产品定义

### 2.1 产品愿景

> **让每一位网文作者都能拥有一个AI编辑团队**——帮你守住质量底线、预判读者反应、规划商业化路径。

### 2.2 产品名称（暂定）

**NovelScope** — 小说望远镜，寓意"看得更远、写得更准"

### 2.3 一句话定位

面向中文网文作者的AI写作质量评估与商业化辅助平台，通过追读力分析、爽点检测、一致性审查和数据驱动的商业化建议，帮助作者写出"能留住读者、能变现"的高质量网文。

### 2.4 核心功能矩阵

#### 模块一：追读力分析系统（核心差异化）

| 功能 | 描述 | 竞品对比 |
|------|------|----------|
| 爽点密度检测 | 分析每章爽点分布密度，标注"爽点真空区" | 仅webnovel-writer有基础版 |
| Hook钩子评估 | 评估章末悬念钩子强度，预测读者翻页率 | 无竞品 |
| 节奏曲线可视化 | 生成全书情绪节奏曲线，标注高潮/低谷/平淡区 | 无竞品 |
| 黄金三章评分 | 专项评估前三章的吸引力指标 | 无竞品 |
| 追读留存预测 | 基于结构化数据预测读者追读留存率 | 无竞品（蓝海） |
| 伏笔债务追踪 | 追踪已埋伏笔的兑现状态，防止"挖坑不填" | webnovel-writer有类似功能 |

#### 模块二：写作质量审查系统

| 功能 | 描述 | 竞品对比 |
|------|------|----------|
| 逻辑一致性检查 | 检测人物设定、时间线、世界观的前后矛盾 | webnovel-writer/大神AI有类似功能 |
| 叙事注水检测 | 识别冗余描写、无效对话、拖沓段落 | 无竞品 |
| 角色声音一致性 | 检测角色对话风格是否前后统一 | 无竞品 |
| 文风稳定性评估 | 分析全文文风波动，标注风格突变点 | 无竞品 |
| AI痕迹检测 | 检测并标注AI生成痕迹，辅助人工润色 | 笔灵AI有类似功能 |

#### 模块三：记忆管理系统

| 功能 | 描述 | 竞品对比 |
|------|------|----------|
| 角色档案自动维护 | 自动提取和更新角色信息（性格/外貌/能力/关系） | webnovel-writer/蛙蛙有类似功能 |
| 世界观知识图谱 | 构建和维护世界观设定图谱 | 无竞品（差异化） |
| 剧情线追踪 | 多条剧情线的并行追踪和交叉检测 | 无竞品 |
| 写前记忆注入 | 写新章节前自动加载相关上下文 | webnovel-writer有此功能 |
| 合同式约束 | 核心设定不可违背，变更需审批 | webnovel-writer独创 |

#### 模块四：商业化辅助系统

| 功能 | 描述 | 竞品对比 |
|------|------|----------|
| 题材热度分析 | 基于市场数据的题材热度趋势和推荐 | 笔灵AI有浅层版本 |
| 平台适配建议 | 分析作品风格，推荐最适合的签约平台 | 无竞品 |
| IP改编潜力评估 | 评估作品的短剧/动漫/影视改编潜力 | 无竞品（蓝海） |
| 签约概率预测 | 基于质量评分和市场数据预测签约成功率 | 无竞品（蓝海） |
| 收入模型模拟 | 模拟不同平台的预期收入模型 | 无竞品（蓝海） |

#### 模块五：AI辅助创作（基础能力，非差异化）

| 功能 | 描述 |
|------|------|
| 智能续写 | 基于记忆注入的上下文感知续写 |
| 扩写润色 | 段落扩写、文风润色、节奏调整 |
| 大纲生成 | 基于题材模板的大纲/细纲生成 |
| 角色设定生成 | 一键生成角色详细设定卡 |

### 2.5 产品架构图（文字描述）

```
用户输入（文本/大纲/设定）
        |
        v
  Next.js App (Vercel)
  ├── 前端 UI (React + Tailwind CSS)
  ├── API Routes
  │   ├── /api/evaluate（评估入口）
  │   │   ├── 规则引擎（本地：爽点密度/节奏/注水）
  │   │   └── DeepSeek API（LLM：Hook/悬念/一致性）
  │   └── 用户/章节/报告 CRUD
  ├── 记忆管理层
  │   ├── 角色档案
  │   ├── 世界观图谱
  │   └── 合同约束
  └── Prisma ORM → PostgreSQL (Supabase)
        |
        v
  质量报告 + 商业化建议
```

| 维度 | 描述 |
|------|------|

### 3.2 用户"一天"使用场景

**日更作者的典型工作流**：

```
09:00  打开NovelScope，查看今日写作任务
09:05  系统自动加载前文记忆（角色/剧情/世界观）
09:10  基于大纲开始创作，AI辅助续写
10:30  完成一章初稿，提交质量审查
10:35  系统返回审查报告：
       - 逻辑一致性：92分（发现1处角色设定冲突）
       - 爽点密度：中等（第3段建议加强冲突）
       - 章末Hook：弱（建议增加悬念）
       - 叙事注水：第5段可精简
10:40  根据反馈修改，重新审查通过
10:50  发布章节，系统自动更新记忆库
11:00  查看全书追读力仪表盘，了解整体节奏
```

## 第四章 竞品分析

### 4.1 竞品全景图

| 竞品 | 定位 | 核心优势 | 核心劣势 | 与我们的关系 |
|------|------|----------|----------|-------------|
| **星月写作** | 网文量产工具 | 出稿速度快、模板丰富、门槛低 | 长篇管理弱、无质量分析 | 互补：我们做质量，它做量产 |
| **webnovel-writer** | 长篇质量控制 | 记忆管理最强、合同系统创新 | 门槛极高、需Claude Code | 直接竞品：我们降低门槛+增加商业化 |
| **蛙蛙写作** | 全链路IP开发 | 小说-剧本-视频全链路 | 追读力分析弱、长篇管理一般 | 部分竞合：我们做深度分析，它做多形态 |
| **笔灵AI小说** | 网文创作工作台 | 功能全面、模板多 | 质量分析浅层、定价偏高 | 竞争：我们在质量分析上更深 |
| **阅文妙笔/Claw** | 平台内置AI | 拥有平台真实数据 | 仅限阅文生态 | 差异化：我们跨平台+独立第三方 |
| **番茄AI写作** | 平台内置AI | 功能丰富、免费 | 受AI治理政策限制 | 差异化：我们独立于平台 |

### 4.2 我们的差异化壁垒

| 差异化维度 | 具体优势 | 竞品无法复制的理由 |
|------------|----------|-------------------|
| **追读力预测模型** | 基于结构化数据的定量追读力分析 | 需要大量网文质量数据积累 |
| **网文专项质量体系** | 专为网文设计的质量评估标准 | 通用AI工具缺乏网文领域知识 |
| **商业化分析引擎** | IP潜力评估+收入模型模拟 | 需要市场数据和行业洞察 |
| **低门槛+深度能力** | webnovel-writer的深度+星月的易用性 | 需要同时具备产品和技术能力 |
| **跨平台独立性** | 不绑定任何网文平台 | 平台内置工具天然受限 |
