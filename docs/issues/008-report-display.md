# Issue #8: 完整报告展示

## What to build

构建完整的评估报告展示组件，包括各项子分数、亮点分析、改进建议和一致性检查结果。

## Acceptance criteria

- [x] 子分数展示：Hook、爽点密度、章末悬念、节奏各 0-10 分（颜色编码：绿 7-10 / 黄 5-6 / 红 0-4）
- [x] 亮点分析：正面反馈优先展示，建设性语气
- [x] 改进建议：使用"建议"而非"问题"，"可以提升"而非"不好"
- [x] 一致性检查结果：逻辑矛盾列表（如有）
- [x] 评分低于 5 分时必须附带具体建议，不显示裸分数
- [x] 空状态：引导卡片 + 图标 + 行动提示
- [x] 部分失败：黄色横幅 + 可用的部分结果
- [x] 错误状态：红色卡片 + 图标 + 重试按钮
- [x] 报告语气为"编辑给作者的反馈"，不是"考试成绩单"
- [x] 组件测试：各状态渲染（success/partial/error/empty）
